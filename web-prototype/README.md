# Feather Detective Web Prototype

网页版方向：把 Unity 里的自由移动探索，收敛成一个 10-15 分钟的点击式网页推理原型。玩家不再控制小鸟移动，而是在公园地图、区域插画、羽毛记忆卡和推理板之间切换，通过阅读、比较和放置线索完成故事。

## Core Idea

玩家是一只小鸟，正在调查为什么公园里的老人突然不再喂鸟。

网页版本保留三个核心体验：

- 点击公园里的羽毛，获得不同鸟类留下的记忆。
- 比较不同鸟类只关注一种现实侧面的线索。
- 在鸟巢推理板上整理羽毛，拼出温柔的真相。

网页版本降低这些部分：

- 不做 3D 移动、跳跃、滑翔和碰撞。
- 不做复杂物理交互。
- 不要求玩家在地图中精准操作角色。

## Player Loop

1. 进入首页，看到标题、主鸟和一句悬念。
2. 打开公园地图，看到 5 个可探索区域。
3. 点击区域，进入该区域的插画场景。
4. 在区域中点击可见羽毛。
5. 羽毛打开记忆卡，展示声音、颜色、路线、节奏或氛围线索。
6. 羽毛进入线索册。
7. 收集足够线索后进入 Nest Board。
8. 把关键羽毛放入 5 个推理槽。
9. 全部正确后解锁结局页。

## Page Flow

```text
index.html
  -> map.html
      -> location-bench.html
      -> location-fountain.html
      -> location-notice.html
      -> location-pond.html
      -> location-rooftop.html
  -> journal.html
  -> board.html
  -> ending.html
```

实现时可以先做成单页应用，用不同 panel 模拟这些页面；文档中的文件名表示体验结构，不强制拆成多个 HTML 文件。

## Pages

### 1. Title Page

目的：给玩家建立情绪和任务。

内容：

- 标题：羽间探忆
- 副标题：一只鸟的推理之旅
- 背景：城市公园远景或主鸟角色图。
- 简短引导：老人已经三天没有来长椅边撒下种子。公园里的鸟们只记得事情的一部分。
- 按钮：开始调查

交互：

- 点击开始后进入公园地图。

推荐素材：

- 主鸟三视图素材用于标题角色。
- 公园街区远景素材作为背景。

### 2. Park Map

目的：作为主要导航页。

内容：

- 俯视公园地图。
- 5 个探索区域以圆形图标标记。
- 顶部显示收集进度：羽毛 0/12。
- 右下角入口：线索册、Nest Board。

区域：

- 长椅 Bench：老人平时喂鸟的位置。
- 喷泉 Fountain：声音、水声、电话声相关线索。
- 告示牌 Notice Board：医院方向、颜色、文字线索。
- 池塘 Pond：鸽子路线和群鸟等待。
- 屋顶 Rooftop：额外氛围线索和误导线索。

交互：

- 点击区域进入对应区域页。
- 已探索区域变亮。
- 区域内所有羽毛收集后显示小勾。

推荐素材：

- 俯视公园地图素材用于主导航。

### 3. Location Pages

目的：让玩家在插画中找羽毛。

结构：

- 大图背景：该区域的公园插画。
- 羽毛热点：用微光、轻微浮动、轮廓线提示。
- 区域标题和一行氛围文字。
- 返回地图按钮。
- 当前已发现羽毛列表。

交互：

- 点击羽毛热点，打开 Memory Card。
- 已点击羽毛保留在画面上，但颜色变淡，避免重复误会。
- 鼠标悬停时显示一句短提示，例如“有一根深色羽毛卡在长椅缝里”。

五个区域安排：

| 区域 | 主视觉 | 羽毛数量 | 叙事功能 |
| --- | --- | ---: | --- |
| 长椅 | 喂鸟长椅、种子碗、鸽子 | 3 | 建立老人消失的问题 |
| 喷泉 | 水声、脚步、电话 | 2 | 引出医院相关声音 |
| 告示牌 | 公园公告、医院方向牌 | 2 | 给出颜色和文字线索 |
| 池塘 | 鸟群、水边路线 | 3 | 给出移动路线和等待氛围 |
| 屋顶 | 城市边缘、误导性亮色 | 2 | 提供非关键线索和世界感 |

### 4. Memory Card

目的：替代 Unity 中的记忆效果。

每根羽毛打开一张记忆卡。卡片不直接说答案，而是只呈现该鸟会注意到的现实侧面。

记忆类型：

| 鸟类 | 网页表现 | 玩家应推断 |
| --- | --- | --- |
| Crow | 暗色卡片、声波线、短音频文字 | 乌鸦只记得声音 |
| Magpie | 灰阶插画，关键颜色高亮 | 喜鹊只记得颜色 |
| Pigeon | 白色路线线条覆盖地图 | 鸽子只记得移动路线 |
| Woodpecker | 节拍点、撞击时间轴、身体动作词 | 啄木鸟只记得节奏和冲击 |
| Sparrow | 风、虫鸣、鸟群紧张度描述 | 麻雀只记得氛围 |

交互：

- 点击“收进线索册”后关闭卡片。
- 已收集羽毛可在 journal.html 重新查看。
- 第一次遇到某鸟类时不解释规则，只让玩家从卡片风格中慢慢发现。

### 5. Clue Journal

目的：让玩家回看和比较线索。

内容：

- 按鸟类筛选：乌鸦、喜鹊、鸽子、啄木鸟、麻雀。
- 按区域筛选：长椅、喷泉、告示牌、池塘、屋顶。
- 羽毛卡片网格：图标、名称、来源区域、记忆摘要。
- 一块“观察札记”：玩家可以看到系统生成的中性提示，例如“有些羽毛描述的是同一天的不同侧面”。

交互：

- 点击羽毛卡再次打开 Memory Card。
- 点击“送到推理板”进入 board.html，并预选这根羽毛。

### 6. Nest Board

目的：最终推理页面。

视觉：

- 软木板背景。
- 便签纸作为 5 个推理槽。
- 左侧是已收集羽毛卡片。
- 右侧是当前选中羽毛的记忆卡摘要。
- 红线连接已放置的羽毛和中央事件照片。

推理槽：

- 最后一次喂食是什么时候？
- 发生了什么变化？
- 他去了哪里？
- 鸟群为什么还在等待？
- 这些证据意味着什么？

交互：

- 点击左侧羽毛，选中。
- 点击推理槽，把羽毛贴上去。
- 槽位已有羽毛时，再点击可替换。
- 点击槽位里的羽毛可取下。
- 放错允许保留，但槽位边缘保持普通颜色。
- 放对后槽位边缘出现柔和金色描边。
- 五个槽全部正确后出现“整理完成”按钮。

不要出现：

- 不要用选择题告诉玩家答案。
- 不要在羽毛上写“这是正确答案”。
- 不要强制线性顺序，玩家可以先放任何槽。

### 7. Ending Page

目的：完成温柔反转。

内容：

- 背景：黄昏公园长椅。
- 画面：鸟群仍在等，长椅上留着一个旧种子袋。
- 文本逐段出现：
  - 鸽子的路线不是逃离，而是反复通往医院方向。
  - 喜鹊记住的蓝白颜色，是医院标识和病房手环。
  - 乌鸦听见的声音，是轮椅、电话和轻声安慰。
  - 老人没有抛弃鸟群。
  - 他只是留在医院，陪着妻子。
- 最后一行：几天后，长椅边又落下一小把种子。

交互：

- 返回线索册。
- 重新整理推理板。

## Asset Plan

用户提供的素材可用于以下位置：

- 五栏鸟类记忆视觉：用于 Memory Card 的鸟类规则说明和视觉参考。
- 主鸟三视图：用于标题页、线索册头像、结局页角色。
- 俯视公园地图：用于 map.html 主导航。
- 公园街景：用于标题页背景、长椅/喷泉区域背景。

实际实现时建议将素材放入：

```text
web-prototype/assets/reference/
web-prototype/assets/backgrounds/
web-prototype/assets/characters/
web-prototype/assets/ui/
```

## MVP Scope

第一版网页原型只做这些：

- 静态首页。
- 一张可点击公园地图。
- 5 个区域面板。
- 12 根可点击羽毛。
- 记忆卡弹窗。
- 线索册。
- 可点击放置的 Nest Board。
- 正确解谜后出现结局页。

第一版暂不做：

- 账号、存档、后端。
- 复杂动画。
- 真正音频制作。
- 多结局。
- 移动端深度适配。

## Implementation Preference

推荐技术路线：

- HTML + CSS + JavaScript 单页原型。
- 数据写在一个 `feathers` 数组里。
- 不引入复杂框架。
- 可以直接本地打开，也可以用简单静态服务器预览。

这个方向比 Unity 更轻，适合快速做出完整体验，也更容易接近参考网页的点击式叙事推理节奏。
